03 问题的核心性质 (Submodularity)

子模性：破解组合爆炸的关键钥匙

难点 (Challenge)

NP-Hard 组合优化问题

从 N 个候选点中选择 K 个，解空间呈爆炸式增长，穷举法完全失效。

Search Space Size

\(\binom{N}{K}\)

Exponential Growth

突破口：子模性 (Submodularity)

Target Function

\( f(S) = \log \det (\mathbf{F}_S) \)

目标函数被证明是单调子模函数 (Monotone Submodular Function)。

收益递减 (Diminishing Returns)

随着震源数量增加，新震源带来的边际信息增益逐渐减少。

重要意义 (Significance)

子模性保证了贪心算法的高效性，并提供了理论最优下界。

04 求解算法 (Lazy Greedy / CELF)

利用边际增益递减特性实现极速求解

lazy_greedy.py 
while len(selected) < K:
    # 1. 取出期望值最高的候选点 (Max-Heap)
    candidate = queue.pop_top() 
    
    if candidate.is_stale:
        # 2. 重新计算真实增益 (昂贵操作)
        candidate.gain = recompute(candidate)
        candidate.is_stale = False
        
        # 3. Look Ahead 检查
        top_of_queue = queue.peek() 
        
        if candidate.gain >= top_of_queue.gain:
            # 即使重算也是第一 -> 选中
            select(candidate)
        else:
            # 比第二名的上限还小 -> 放回队列重排
            queue.push(candidate) 
            # 循环继续，下一次考察原第二名
    else:
        # 已经是新鲜值 -> 直接选中
        select(candidate)

核心逻辑

  • 增益上限性质：

    队列中的“旧增益”永远是真实增益的上限 (Upper Bound)。

  • 懒惰计算 (Lazy Evaluation)：

    只有当“第一名”的地位受到威胁时，才重新计算它的真实价值。

时间复杂度

令 Cost 为单次目标函数评估耗时（最耗时部分）。

Standard Greedy
\( \mathcal{O}(K \cdot N \cdot \text{Cost}) \)

每一轮都要重新计算剩余所有 \(N\) 个候选点的增益。计算量巨大。

Lazy Greedy (CELF)
\( \mathcal{O}(N \cdot \text{Cost} + K \cdot \bar{n} \cdot \text{Cost}) \)

仅初始化需 \(N\) 次计算。后续每轮平均仅需重算 \(\bar{n}\) 个点 (\(\bar{n} \ll N\))。

实测差异： \( K \cdot N \gg N + K \cdot \bar{n} \)

08 算法效率分析 (Computational Efficiency)

时间复杂度分析

考察当选择数量 K 从 10 增加到 30 时，不同算法的计算耗时变化。

关键发现

  • Lazy Greedy

    极其高效。耗时几乎随 \(K\) 线性微增。惰性机制成功避免了大量冗余计算。

  • Local Search

    耗时略高，但仍在可接受的秒级范围内。

Conclusion

贪心策略在工程实时性上具有压倒性优势。

10 总结 (Conclusion)

项目回顾
