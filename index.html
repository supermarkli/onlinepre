01 背景：eBPF 核心机制 (Background)

什么是 eBPF？(What is eBPF?)

eBPF 是一种革命性的内核技术，允许在不修改源码的情况下，将沙箱化的用户程序挂载到内核执行路径中。

拥塞控制 (Context)

传统 TCP 算法依赖内核模块硬编码，eBPF 为其赋予了动态可编程性，是实现“慢思考”架构的基石。

挂载点 Hooks

挂载点
像“血管夹”一样，在 TC, XDP 等关键路径精准拦截流量。

验证器 Verifier

验证器
内核的“门卫”，确保程序无死循环、无越界，保障 OS 绝对安全。

状态存储 Maps

状态存储
跨越内核/用户态的高效 Key-Value 存储，数据交互的桥梁。

辅助函数 Helpers

辅助函数
白名单能力接口，提供获取时间、修改包头等原子操作。

02 现状：eBPF 与 ICC 生态 (Landscape)

eBPF 已成事实标准 Industry Standard

云原生与安全 (Success Stories)

Meta Katran (L4 LB) 与 Cilium (Security) 利用 eBPF 实现了无内核补丁的高性能转发，已在大规模生产环境中验证了绝对统治力。

ICC 领域的当前定位: “辅助支撑”

深度观测 (Observability)

透视 TCP 内部状态 (RTT, cwnd)，作为“显微镜”辅助决策。

动态编排 (Orchestration)

动态调整传统内核参数 (sysctl) 或切换算法，非直接接管执行。

智能拥塞控制 (ICC) 三大流派
学习驱动 (Learning-driven)
DeepCC [Sigcomm'20]
核心思路：利用 DRL 学习帕累托最优策略集。
局限性：模型推理开销大，难以应对高并发短流。
遥测驱动 (Telemetry-driven)
TCP's Third Eye [NSDI'23]
核心思路：利用 eBPF 消费最拥塞跳的 INT 遥测数据。
局限性：依赖 TCP 选项字段，严重破坏 GRO 机制导致吞吐下降。
启发式与可编程 (Heuristic)
eTran [NSDI'24]
核心思路：在 XDP 层实现定制化传输协议 (如 Homa)。
局限性：需修改内核源码添加钩子，非标准 eBPF 路径。
Reality Gap

尽管学术界百花齐放，但 eBPF 目前在 ICC 中多仅用于 旁路观测 或 动态配置。

旗舰算法 (Google BBR) 仍依赖原生内核模块，尚未实现真正的“内核级智能执行”。

03 核心挑战 (Challenges)

智能算法与 eBPF 执行路径存在天然鸿沟，形成 “认知-执行错配”

计算能力鸿沟

智能算法需要浮点与复杂推理，eBPF 禁止浮点且指令受限，深度模型难以内核态运行。

可见性代价

为获取细粒度信号修改包头会破坏 GRO，吞吐下滑；逐包遥测的 CPU/缓存开销难以接受。

外部控制延迟

用户态控制回路往返可达秒级，无法应对微秒级拥塞；内核态缺乏低延迟策略更新通路。

模型直搬不可行

直接将深度学习模型搬进内核不可行，当前工业落地率极低，易导致可靠性与运维风险。

核心思路：将“思考”(AI 推理) 与“执行”(数据面动作) 彻底解耦

04 系统整体架构 (System Architecture)

System_Architecture

架构创新：异步双平面架构

现有智能拥塞控制采用同步闭环模式，控制回路时延包含推理耗时以及用户态与内核态的交互。

规避了 eBPF 的计算约束，实现了“零推理时延”的拥塞响应
控制平面更新频率低至 20Hz (50ms)，数据平面仍能保持 Per-ACK 级别的微秒级响应精度

慢思考平面 (User Space)

高维决策与策略生成
  • AI Agent (大脑): 运行 DRL 模型，处理历史趋势与全局梯度。
  • Map Generator: 将推理结果转化为结构化策略表 (Policy Table)。
  • 异步闭环: 10-50ms 周期轮询，不阻塞数据通路。
Data Uplink Polled Stats (无锁拉取)
Strategy Downlink Map Push (异步更新)

快执行平面 (Kernel Space)

微秒级响应与精准执行
  • eBPF (反射神经): 拦截 TCP 事件，基于Log（RTT）+梯度进行O（1）查表。
  • Socket Storage: 存储 Per-Flow Maps，实现流间策略隔离。
  • EDT (起搏器): 利用 Linux原生调度器实现纳秒级 Pacing。
  • 安全熔断: 监测到严重丢包时，直接旁路 AI 策略进行降级保护。

05 关键创新：策略投影与高效索引 (Projection & Indexing)

算法创新：高维策略投影 (Map-Driven Projection)

机制：AI 输出向量场 (Vector Field)而非单一标量。预测在各种 RTT 与梯度组合下的最佳速率。

核心价值：将 NN 非线性推理降维为内核 O(1) 查表，兼顾智能决策与 eBPF 轻量化要求。

数据结构：对数分箱与复合索引 (Log-Binning)

对数分箱: 利用 bpf_log2() 压缩 RTT。低延迟区高精度，长肥管道宽覆盖。
复合索引: Index = (Log_RTT << 2) | Trend
核心价值：极低内存开销 (每流仅几百字节) 实现广域网宽动态范围覆盖与趋势感知。

06 关键创新：生命周期与鲁棒性 (Lifecycle & Robustness)

Lifecycle_Workflow
全生命周期混合编排

分阶段治理: 启动期信任 HyStart++ 快速爬升，稳定期移交 AI。

无缝软切换: 引入加权融合窗口，消除算法切换震荡。

Ratefinal = α(t)·RateKer + (1-α)·RateAI
无锁通道与 EDT 集成

无锁拉取: 采用 Per-CPU Array 替代 Ring Buffer，吞吐量随核数线性扩展。

原生 EDT: 依托 Linux 调度器，eBPF 仅设速率，实现零阻塞纳秒级发包。

脊髓反射式熔断

机制: 检测到严重丢包/恢复时，立即旁路 (Bypass) AI，执行硬编码降速。

价值: 无论 AI 如何失效，兜底机制保障协议栈的工业级鲁棒性。

07 工程实现与可行性 (Engineering & Feasibility)

内核集成 (struct_ops)

利用 Linux 5.6+ 特性，无缝替换 TCP 拥塞控制钩子，无需 Patch 内核源码，即插即用。

对数分桶 (Logarithmic Binning)

利用 bpf_log2() 压缩 RTT 索引，兼顾低延迟精度与广域网覆盖，极大节省 Map 内存占用。

极致低开销

单流表仅 512 Bytes，轻松支持 10w+ 并发。使用 Map Polling 替代高开销的 Ring Buffer。

08 总结与展望 (Conclusion)

架构解耦

打破传统束缚，实现 "慢思考" (用户态 AI 决策) 与 "快执行" (内核态 eBPF 调度) 的完美分离。

O(1) 极速执行

拒绝内核浮点运算。通过 查表法 (Map-driven) 将复杂的推理转化为纳秒级的内存读取，零额外开销。

生产级可用

内置 "脊髓反射" 熔断机制。即使 AI 失控，底层 eBPF 也能依据物理网络反馈兜底，保障系统不崩。
